## Design Context

### Users

nav2md is for public users who need to collect technical documentation at scale. They use the Chrome extension on documentation sites to select navigation links, export chapters to Markdown, and prepare clean source material for AI/RAG knowledge bases.

### Brand Personality

The interface should feel clear, easy to use, and professional. It should help users feel confident that they can select the right pages, understand the current export state, and move quickly without learning a heavy tool.

### Aesthetic Direction

Use a refined product feel rather than a developer-debugger look. Keep the current teal brand color direction and favor a light, polished floating panel that remains legible on both light and dark documentation pages. The UI should feel like a mature SaaS utility: compact, calm, precise, and visibly designed without becoming decorative.

Avoid visual directions that are too flashy, too dark, overly animated, or reminiscent of DevTools. Motion should be minimal and purposeful.

### Design Principles

- Prioritize clarity in small spaces: labels, counts, status text, and shortcut hints must be scannable at panel size.
- Preserve user focus on the page: the panel should support selection and export without dominating the document.
- Use the existing teal accent sparingly for state, action, and selection feedback.
- Prefer polished product details over technical chrome: soft borders, crisp spacing, readable keycaps, and restrained surfaces.
- Maintain accessible basics by keeping contrast clear, controls keyboard-readable, motion minimal, and state not dependent on color alone.
